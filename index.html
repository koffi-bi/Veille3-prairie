Comprendre les API

Fonctionnement, exemples, langages et analyse

1. Comment fonctionne une API ?

Une API (Application Programming Interface) permet à deux systèmes de communiquer.

Processus :

1- L’utilisateur fait une action (clic, formulaire...)
2- Le frontend envoie une requête HTTP
3- L’API traite la demande
4- Le serveur renvoie une réponse (souvent en JSON)
5- Le frontend affiche les données

Schéma :

Utilisateur → Frontend → API → Serveur → Base de données → Réponse JSON → Interface

2. Exemples d’API existantes

API météo
Donne la météo en temps réel.
API paiement (Wave, Orange Money,Moov Money,Mtn Money, PayPal...)
Permet d’effectuer des paiements sécurisés.
API Google Maps
Affiche des cartes interactives.
API réseaux sociaux
Permet de publier ou récupérer des données.

3. Comment les langages interagissent avec une API ?

JavaScript
Utilise fetch() ou axios pour envoyer des requêtes HTTP.
Python
Utilise la bibliothèque requests.
PHP
Utilise cURL pour envoyer des requêtes.
Java
Utilise des bibliothèques comme HttpClient.

Tous ces langages envoient des requêtes HTTP (GET, POST, PUT, DELETE) et reçoivent des réponses JSON.

4. Les API les plus utilisées

API REST
Basée sur HTTP. Très populaire et simple.

Forces : simple, standard, largement adoptée

Faiblesses : parfois trop de requêtes

GraphQL
Permet de demander exactement les données nécessaires.

Forces : flexible, optimise les requêtes

Faiblesses : plus complexe à mettre en place

SOAP
API plus ancienne, basée sur XML.

Forces : très sécurisée

Faiblesses : lourde et complexe

5. Forces et faiblesses générales des API

Permettent l’intégration entre systèmes

Gagnent du temps de développement

Réutilisation de services existants

Dépendance à un service externe

Problèmes de sécurité possibles

Limites de requêtes (rate limit)
