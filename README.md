Présentation API
C'est quoi une API ? (Définition simple)
• L’exemple du restaurant :
• Le Client (Vous) : L'application qui veut une donnée.
• La Cuisine (Serveur) : Le système qui détient la donnée.
• Le Serveur (L'API) : Le messager qui prend votre commande et apporte le 
plat.
Imaginez que l'API est le serveur dans un restaurant : vous (l'utilisateur) 
donnez votre commande au serveur (l'API), il l'apporte en cuisine (le système 
distant), et il revient avec votre plat (la donnée ou le service).
• En bref : C'est un contrat qui définit comment deux logiciels communiquent.
À quoi ça sert ?
• Gagner du temps : Utiliser des fonctionnalités existantes au lieu de les coder (ex: 
Maps, Paiement).
• Automatiser : Faire circuler l'information entre les outils de l'entreprise.
        • Sécuriser : Accéder aux données sans exposer tout le système.
Comment ça marche ? (Le cycle)
1. La Requête (Request) : L'application demande une info (ex: GET /meteo/
paris).
2. Le Traitement : L'API vérifie les droits et cherche l'info.
Une API fonctionne selon un cycle simple de Requête et Réponse : l'Appel 
envoie votre demande (ex: la météo), le Traitement vérifie vos droits et 
cherche l'info, puis le Retour vous livre la donnée brute au format JSON.
La Réponse (Response) : L'API renvoie les données (souvent en format JSON).
Les API dans notre quotidien
• Connexion sociale : "Se connecter avec Google/Facebook".
• Paiement : Payer par carte via Stripe ou PayPal sur un site marchand.
• Météo : Les widgets sur votre téléphone qui tirent les données de Météo France.
• Voyage : Les comparateurs (Skyscanner) qui interrogent les API des compagnies aériennes.
Les formats et standards
• REST : Le plus commun, utilise le Web (HTTP) et le format JSON.
• SOAP : Plus ancien, très sécurisé, utilise le format XML.
• GraphQL : Moderne, permet de demander "juste ce qu'il faut" pour aller plus vite.
Pourquoi est-ce vital pour le Business ?
• Économie d'API : Transformer son service en produit que d'autres peuvent 
acheter.
• Innovation : Permet de créer de nouveaux services en combinant plusieurs API 
(Mashups).
• Agilité : Mise à jour simplifiée des systèmes complexes.
Les API boostent le business via l'économie d'API, qui transforme un service en produit 
monétisable. Elles stimulent l'innovation par la fusion de services (Mashups) et apportent une 
réelle agilité en simplifiant l'évolution des systèmes complexes
